В данном форке Sketchware Pro переведёт на 90% процентов на русский язык для поддержки русский разработчиков мобильных приложений через Sketchware Pro.
Релизы будут максимально быстро адаптироватся под главный Git Hub репозиторий, то есть при выпуске новой версии Sketchware Pro, сразу же будут переводится новые функции.

## Building the App
Чтобы создать приложение, вы должны использовать Gradle. Настоятельно рекомендуется использовать Android Studio для обеспечения наилучшего взаимодействия.

### Source Code Map

| Class           | Role                                        |
| --------------- | ------------------------------------------- |
| `a.a.a.ProjectBuilder`      | Помощник для компиляции всего проекта       |
| `a.a.a.Ix`      | Ответственный за создание AndroidManifest.xml |
| `a.a.a.Jx`      | Генерирует исходный код действий          |
| `a.a.a.Lx`      | Генерирует исходный код компонентов, таких как прослушиватели и т.д. |
| `a.a.a.Ox`      | Отвечает за генерацию XML-файлов макетов |
| `a.a.a.qq`      | Реестр зависимостей встроенных библиотек |
| `a.a.a.tq`      | Ответственный за составление тестов в диалоговом окне |
| `a.a.a.yq`      | Упорядочивает пути к файлам проектов Sketchware    |

> [!TIP]
> Вы также можете ознакомиться с пакетом 'mod', который содержит большинство изменений, внесенных авторами.

## Способствующий

Если вы хотите внести свой вклад в Sketchware Pro, выполните следующие действия:

1. Разветвите этот репозиторий.
2. Внесите изменения в свой разветвленный репозиторий.
3. Протестируйте эти изменения.
4. Создайте запрос на загрузку в этом репозитории.
5. Ваш запрос на обновление будет рассмотрен участниками репозитория и объединен, если он будет принят.

Мы приветствуем вклады любого размера, будь то основные функции или исправления ошибок, но, пожалуйста, обратите внимание, что все вклады будут тщательно проверены.

### Сообщение о фиксации

Когда вы вносите изменения в один или несколько файлов, вам необходимо зафиксировать эти изменения с помощью сообщения о фиксации. Вот несколько рекомендаций:

- Сообщение о фиксации должно быть коротким и подробным.
- Используйте один из этих типов фиксации в качестве префикса:
  - `feat:` для какой-то функции, возможно, улучшающей что-то уже существующее.
  - `fix:` для исправления, например, ошибки.
  - `style:` для получения информации о функциях и обновлениях, связанных со стилем.
  - `refactor:` для рефакторинга определенного раздела кодовой базы.
  - `test:` для всего, что связано с тестированием.
  - `docs:` для всего, что связано с документацией.
  - `chore:` для поддержки кода (вы также можете использовать эмодзи для представления типов фиксации).

Examples:
- `feat: Ускорил компиляцию с помощью новой технологии`
- `fix: Исправлен сбой при запуске на некоторых телефонах`
- `refactor: Переформатировать код в File.java`

> [!ВАЖНО]
> Если вы хотите добавить новые функции, которые не требуют редактирования в других пакетах, кроме "pro.sketchware", внесите изменения в пакет "pro.sketchware", соблюдая структуру каталогов, файлов и их названия. Кроме того, несмотря на то, что проект отлично компилируется с классами Kotlin, которые вы могли бы добавить, старайтесь вносить свои изменения или дополнения на Java, а не на Kotlin, если только это не является чем-то большим, чем необходимо.
